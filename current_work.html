ForkAndMove : Chess Engine with Parallel Aplha-Beta Algorithm [Code]

Bitwise operators are crazy!
I'm building my own chess engine from scratch. It's a fun project which uses bitmaps for efficient coomputation of chess moves and board evaluation using just bitwise operators. I've began to realise it is not as easy as I thought it would be. Rules like en-passant and castling makes it a bit complicated.
I've so far implemented bitmaps implementation of board representation, board evaluation, move generators. My next goal is to write an efficient parallel implementation of widely used aplha-beta search algorithm. I have not found any OpenMP based effcient implementation of this as this algorithm is inherently sequential.
I've got few ideas to proceed with like use of beam search, parallely computing nodes upto certain plie then check values of aplha and beta and maybe resume computation.
I'm implementing in MaPLe, a Standard ML based functional language for provably efficient and safe multicore parallelism.
